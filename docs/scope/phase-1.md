# Phase 1 Scope

## Purpose

Phase 1 is the smallest useful control surface for RelayHQ.

It should be enough to coordinate real work across people and agents without trying to solve the whole product on day one.

## Included

- project registry
- task board
- human and agent assignment
- approvals
- audit notes

## Excluded

- full automation platform
- marketplace for third-party agents
- advanced analytics
- billing

## Why this scope

These features are the minimum needed to coordinate work responsibly:

- projects give structure
- tasks give visibility
- assignment gives ownership
- approvals give control
- audit notes give traceability

Phase 1 stays small so the product can ship in usable slices and grow from real usage instead of speculation.

## What success looks like

Phase 1 is successful when RelayHQ can:

- register a project as the home for related work
- show active tasks clearly
- assign work to a human or an agent
- pause work for approval when needed
- preserve a readable audit trail of decisions and outcomes

## Important boundaries

- RelayHQ is not the agent runtime.
- RelayHQ is not a generic collaboration suite.
- RelayHQ is not a marketplace.
- RelayHQ is not an analytics product.
- RelayHQ should keep coordination explicit and traceable.

## Scope rule

If a feature does not help with project registration, task coordination, assignment, approval, or auditability in Phase 1, it should be deferred unless a decision record says otherwise.
