# Chat

> Future scope: this capability is part of RelayHQ’s long-term vision and is not implemented yet.

## What it is

Chat is a coordination layer for people and agents to discuss work in context.

Example:

- ask for clarification on a task
- request an approval
- hand off work between humans and agents
- capture decisions near the work itself

## Why RelayHQ needs it

RelayHQ needs chat because coordination often happens faster in conversation than in structured forms.

Chat helps:

- reduce context switching
- keep decisions close to the task
- preserve the reason behind changes
- make human/agent handoffs smoother

## Who uses it

- contributors
- reviewers
- project leads
- agents participating in the workflow

## How it fits the control-plane vision

Chat should support the control plane, not replace it.

The goal is not “just messaging.” The goal is:

- conversation tied to work objects
- conversation that can trigger actions
- conversation that creates traceable history

## Boundaries / non-goals

Chat is not:

- a general-purpose Slack clone
- a social feed
- an unstructured inbox
- the primary system of record for work

## Future scope note

This is planned future scope and exists to support coordination, traceability, and agent collaboration.
