# Reminders

> Future scope: this capability is part of RelayHQ’s long-term vision and is not implemented yet.

## What it is

Reminders are time-based prompts that bring attention back to work that needs follow-up.

Example:

- remind me to review this task tomorrow
- ping the owner if no update arrives in 3 days
- surface an approval that is still waiting

## Why RelayHQ needs it

RelayHQ needs reminders because control planes fail when attention drops.

Reminders help:

- prevent stalled work
- keep approvals moving
- reduce forgotten follow-ups
- support reliable handoffs between people and agents

## Who uses it

- project leads
- reviewers
- operators
- agents that need follow-up triggers

## How it fits the control-plane vision

Reminders make RelayHQ proactive.

Instead of only recording work, RelayHQ should help teams:

- notice inactivity
- re-open pending items
- maintain momentum
- keep coordination visible over time

## Boundaries / non-goals

Reminders are not:

- a personal calendar replacement
- a full notification platform
- a marketing automation system
- a generic scheduling engine

## Future scope note

This is planned future scope and should remain tightly connected to work items, approvals, and follow-up states.
