# Assignment

## Purpose

Assignment is how RelayHQ makes ownership explicit.

## Why it exists

Work breaks down when no one knows who is responsible. RelayHQ uses assignment to remove ambiguity for both humans and agents.

## How humans use it

Humans use assignment to:

- assign tasks to people or agents
- reassign work when scope changes
- confirm who is accountable
- hand off work with context

## How agents use it

Agents use assignment to:

- know when a task is theirs
- know when to pause and wait for a human
- hand work back when review or approval is needed
- avoid acting outside assigned scope

## Flow / lifecycle

1. A task is created.
2. An owner is assigned.
3. The assignee works the task or triages it.
4. Ownership may change as the task moves.
5. The task is completed, handed off, or closed.

## Boundaries / non-goals

- not a generic permission system
- not a substitute for role management
- not a shared ownership blur
- not a runtime scheduler for agents

## Traceability / ownership notes

- assignment should be visible in the task record
- reassignment should preserve history
- the current owner and past owners should be clear
- if ownership is unclear, the task is not ready
