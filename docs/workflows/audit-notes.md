# Audit Notes

## Purpose

Audit notes preserve the "why" and "what happened" behind important work.

## Why it exists

RelayHQ is a control plane, so history matters. Audit notes make decisions and agent actions traceable later.

## How humans use it

Humans use audit notes to:

- record important decisions
- explain why a task changed
- document review outcomes
- leave context for future handoffs

## How agents use it

Agents use audit notes to:

- record actions taken
- summarize results
- explain exceptions or failures
- leave a clear trail for human review

## Flow / lifecycle

1. A meaningful action happens.
2. A note is written with context.
3. The note is attached to the relevant project or task.
4. Future readers use it to reconstruct what happened.
5. Notes remain as durable history.

## Boundaries / non-goals

- not a free-form chat log
- not a replacement for status fields
- not a dumping ground for noise
- not meant for high-volume, low-value updates

## Traceability / ownership notes

- notes should be attributable to a person or agent
- notes should reference the related project or task
- keep notes short, factual, and time-sensitive
- the goal is durable evidence, not narration
