# Approvals

## Purpose

Approvals are RelayHQ’s control point for risky or meaningful actions.

## Why it exists

Agent-assisted work often needs human review before something is finalized, published, or applied.

## How humans use it

Humans use approvals to:

- review proposed actions
- accept or reject changes
- add conditions or feedback
- keep responsibility explicit for high-impact steps

## How agents use it

Agents use approvals to:

- request permission before proceeding
- present a clear summary of the intended action
- wait for human response
- continue only when approval is granted

## Flow / lifecycle

1. An action is proposed.
2. The action is submitted for approval.
3. A human reviews the request.
4. The request is approved, rejected, or revised.
5. The result is recorded with context.
6. The related task continues or stops accordingly.

## Boundaries / non-goals

- not every task needs approval
- not a general review system for all content
- not a substitute for trust and good task design
- not a blocker for routine low-risk work

## Traceability / ownership notes

- approval requests should capture what is being approved
- the approver should be recorded
- the reason for the decision should be preserved
- approvals are part of the audit trail, not a separate side channel
