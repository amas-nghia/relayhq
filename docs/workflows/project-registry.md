# Project Registry

## Purpose

The project registry is the system of record for active work in RelayHQ.

## Why it exists

RelayHQ needs a stable place to define what exists before tasks, assignment, approvals, and audit notes can work well.

## How humans use it

Humans use the registry to:

- create projects
- review project scope and status
- find the right home for new work
- confirm ownership and context

## How agents use it

Agents use the registry to:

- understand project context before acting
- link work back to the right project
- avoid operating on orphaned tasks or unclear scope

## Flow / lifecycle

1. A project is created.
2. Basic metadata and ownership are recorded.
3. Tasks, approvals, and notes attach to the project.
4. The project stays as the traceable anchor for all related work.
5. When work ends, the project remains available for history and review.

## Boundaries / non-goals

- not a full project management suite
- not a document repository
- not a planning tool for deep roadmaps
- not the place for operational task execution

## Traceability / ownership notes

- every task should point back to a project
- project ownership should be explicit
- project history should remain readable after work is complete
- the registry is the anchor for auditability
