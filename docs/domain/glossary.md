# Glossary

RelayHQ uses the following terms consistently.

## Project

A project is a named unit of coordinated work.

It groups related tasks, approvals, notes, and history under one traceable container.

## Plan

A plan is a higher-level structure that breaks a goal into ordered or related work items.

Plans help describe how work should unfold before it becomes a set of tasks.

## Task

A task is the atomic unit of work in RelayHQ.

Tasks can be assigned, approved, tracked, and audited.

## Assignment

An assignment states who is currently responsible for a task.

Assignments may point to a human or an agent.

## Human

A human is a person using RelayHQ to coordinate, review, approve, or own work.

## Agent

An agent is a non-human worker that receives tasks, reports progress, requests approval, and returns results.

## Approval

An approval is a traceable decision about whether an action may proceed.

Approvals exist to keep risky or important actions explicit.

## Audit note

An audit note is a durable record of what happened, who did it, and why it matters.

Audit notes are part of the history of the work, not a casual chat stream.

## Progress snapshot

A progress snapshot is a point-in-time summary of work status.

It helps show what is moving, blocked, or complete without reconstructing the full history.

## Reminder

A reminder is a scheduled or triggered nudge that brings attention back to work that needs follow-up.

## Report

A report is a synthesized summary of work for a human audience, often external to the team.

## Thread / Chat

A thread or chat is a conversation linked to work.

It is useful for coordination, clarification, and handoffs.

## Workspace

A workspace is the top-level boundary that contains projects and related coordination state for one team or organization.

## Run / Session

A run or session is a time-bounded instance of work execution or coordination.

The term is useful when a task moves through a specific attempt or interaction history.
